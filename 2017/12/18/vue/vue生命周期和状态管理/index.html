发表于 2017-12-18 | 分类于 vuejs

vue生命周期的实现过程

  1. vue生命周期分为创建状态,更新,销毁三种过程
  2. 在创建过程中,会调用beforeCreate,create,beforeMounte,Mounted这4个钩子函数,其中只有beforeCreate执行的时候没有初始化data和methods中定义的属性和方法,其他三种都可以使用,如果有ajax请求需要放在除beforeCreate外的另外三个钩子函数中
  3. data数据更新过程会触发beforeUpdate(数据更新前调用)和Updated(数据更新后调用)两个钩子函数,同时获取更新后的值
  4. 当vue实例销毁消失的时候(页面跳转的时候)上一个页面的vue销毁触发beforeDestroy和Destroyd,这时候可以执行清除工作

总结

  • Vue构造函数原型中存在了beforeCreate,create,beforeMounte,mounted,beforeUpdated,Updated,beforeDestroy和Destroyd,
  1. 创建Vue的实例的时候会先执行beforeCreate钩子
  2. 之后会监控data和methods属性的变动,此时data和methods初始化完成
  3. create钩子开始执行,但是html页面还没有生成
  4. 判断是否有el属性,有的话就会检查是否有表达式存在将data的数据和模板生成html,但是还没有加载到浏览器中,如果没有则抛出异常
  5. 如果检查不到el属性则会等到有才会执行第4步
  6. 生成HTML之后会执行beforeMount挂载钩子的准备工作
  7. 将编译好的HTML内容替换el属性指向的区域
  8. 执行mounted函数,此时的dom树才算真正渲染完成,可以进行操作,mounted只会执行一次
  9. 进行数据监视,一旦data数据发生变化就会触发beforeUpdated和Updated两个钩子,同时获取到的都是最新的值
  10. 最后执行beforeDestroy和Destroyd函数

VUEX 状态管理

  • Vuex 应用的核心就是 store（仓库）,其实就是全局数据存储,供全组件调用,其状态是响应式的,只要有改变,所有依赖项也会跟着改变

核心概念

State

  • 单一状态树,是唯一数据源 (SSOT),对外是只读属性,无法直接更改,只能用显式地提交 (commit) mutation中的方法来改变state中的数据,可以用store.state获取数据,state可以存储任何类型的数据

    Getter

  • 可以认为是 store 的计算属性,getter 的返回值会根据它的依赖被缓存起来，且只有当它的依赖值发生了改变才会被重新计算

    Mutation

  • 改变store的唯一方法,每个 mutation 都有一个字符串的 事件类型 (type) 和 一个 回调函数 (handler),回调就是处理store数据变化的回调,第一个参数为state,第二个参数是载荷（payload）,通常传入对象,可以包含较多数据,即用store.commit(type,payload)调用,payload={key.val},

    Action

  • Action 提交的是 mutation，而不是直接变更状态。Action 可以包含任意异步操作。使用store.dispatch(Mutation里的type)触发,也支持载荷（payload）

    Module

  • 对store进行模块管理,每个模块拥有自己的 state、mutation、action、getter、甚至是嵌套子模块——从上至下进行同样方式的分割
